1) Project Theme & Tokens
- Primary intent: warm, welcoming café aesthetic. Use amber/gold accents with calm neutrals.

Color palette (tokens)
- `--color-bg`: #fff7ed (warm cream) / light background
- `--color-card`: #fff (cards)
- `--color-primary`: #b45309 (amber-700) — buttons, accents
- `--color-accent`: #f59e0b (amber-500)
- `--color-muted`: #6b7280 (gray-500)
- `--color-text`: #111827 (neutral-900)
- `--color-success`: #16a34a (green-600)
- `--color-danger`: #dc2626 (red-600)

Type scale
- Base font-size: 16px
- Scale: `xs:12px`, `sm:14px`, `base:16px`, `lg:18px`, `xl:20px`, `2xl:24px`, `3xl:30px`, `4xl:36px`

Spacing scale
- Use Tailwind spacing scale (1 => 0.25rem). Prefer multiples: `p-4`, `py-6`, `gap-4`.

Border radius
- `rounded-sm: 0.125rem`, `rounded: 0.25rem`, `rounded-md: 0.375rem`, `rounded-lg: 0.5rem`.

Shadows
- Subtle card shadow: `shadow-sm` (0 1px 2px rgba(0,0,0,0.05)).
- Elevated components: `shadow-md`.

Iconography
- Use a single icon set (Heroicons or Font Awesome). SVGs preferred for crispness.

2) Tailwind Configuration (recommended)
- Provide a `tailwind.config.js` to centralize tokens.

Example settings (conceptual):
- `theme.extend.colors` -> map the tokens above to names like `primary`, `accent`, `muted`.
- `fontFamily`: `heading: ['Cormorant Garamond', 'serif']`, `body: ['Montserrat', 'sans-serif']`.
- `container`: center and padding config.
- `screens`: default Tailwind (`sm:640`, `md:768`, `lg:1024`, `xl:1280`). Use `2xl` only if needed.

Key plugins
- `@tailwindcss/forms` -> consistent form controls.
- `@tailwindcss/typography` -> prose styling for about pages or blog content.

Example tailwind fragment (for your `tailwind.config.js`):
- `module.exports = { content: ['./src/**/*.{js,jsx,ts,tsx,html}'], theme: { extend: { colors: { primary: '#b45309', accent: '#f59e0b', muted: '#6b7280' }, fontFamily: { heading: ['Cormorant Garamond', 'serif'], body: ['Montserrat', 'sans-serif'] } } }, plugins: [require('@tailwindcss/forms'), require('@tailwindcss/typography')] }`

5) Page-by-Page Layouts (structure + class suggestions + details)
- Each page description includes: header, hero/lead component, content sections, CTAs, footer.

A) Home
- Intent: introduce the café, highlight specials and CTAs for reservations and ordering.
- Layout sections:
  - `TopNav` -> sticky, `bg-white/50 backdrop-blur` on scroll, logo left, actions right (search, cart, profile).
  - `Hero` -> full-width `min-h-[60vh]` with background image, overlay, headline (`text-4xl sm:text-5xl font-heading`), subtitle, CTA buttons (`btn-primary`, `btn-secondary`).
  - `Features Grid` -> 3-4 columns, icons + short text (`grid grid-cols-1 sm:grid-cols-2 md:grid-cols-3 gap-6`).
  - `Popular Menu` -> horizontal scroll on mobile, grid on desktop, each item as `card` with image, name, price, favourite action.
  - `Reviews` -> carousel or grid of `card` blocks.
  - `Footer` -> site nav, contact info, small form for newsletter.

Accessibility & UX notes
- Hero uses `h1` for main heading. CTA buttons have `aria-label`.
- Ensure keyboard access to carousel items.

B) Menu (Listing)
- Intent: browse categories and filter items.
- Layout:
  - `Sidebar` (desktop) -> category list, filters (veg toggle, price range), sticky.
  - `Main` -> search bar (`input`), sort controls, grid of `menu-card`.
  - `menu-card` -> `card` containing lazy-loaded image on top, title, description (truncate to 2 lines), price, add-to-cart button.
- Classes: `grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-6`.
- Behavior: add-to-cart opens small inline drawer or toast; quantity control in cart.

C) Menu Item (Details) / Modal
- Intent: show full description, ingredients, nutrition, images and reviews.
- Layout:
  - `two-column` on medium+: image gallery left (`w-1/2`), details right (`w-1/2`). On mobile stack vertically.
  - `image gallery` -> main image + thumbnails (use `object-cover rounded-lg`).
  - `details` -> title `text-2xl font-heading`, descriptions, price, addons, quantity, CTA `Add to cart`.
- Accessibility: images have `alt` text; thumbnails focusable.

D) About
- Intent: brand story and team.
- Layout:
  - `Hero small` -> headline + mission line.
  - `Two-column story` -> text + supportive image.
  - `Team grid` -> `grid grid-cols-1 sm:grid-cols-2 md:grid-cols-3 gap-6` each team member `card` with photo, role, socials.
- Use Tailwind `prose` classes for story content.

E) Gallery
- Intent: visual showcase.
- Layout: masonry-like grid (`grid-cols-2 md:grid-cols-3 lg:grid-cols-4 gap-3`), lightbox on click.
- Lazy-load thumbnails; load high-res on lightbox open.

F) Contact
- Intent: contact CTA and map.
- Layout: two-column `md:grid-cols-2` -> left: contact form, right: map + contact info + hours.
- Form elements: name, email, message; include success/error feedback and client-side validation.

G) Reservations
- Intent: book a table.
- Layout:
  - Booking widget: datepicker, time slots, guest count, notes.
  - Booking calendar/list view for admin staff to manage.
- Mobile UX: single-column flow with progressive disclosure.

H) Customer Dashboard
- Intent: personal area for orders, reservations, favorites.
- Layout:
  - `Sidebar` (collapsible) with nav links (Profile, Orders, Reservations, Favorites)
  - `Main` -> summary widgets at top (`grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-4 gap-4`), list of recent orders below in responsive table or card list.
- Orders: use status badges with clear colors.

I) Admin Dashboard
- Intent: comprehensive management and reporting.
- Layout:
  - `Left Sidebar` (vertical, collapsible) with clear visual hierarchy.
  - `Topbar` -> quick actions (new menu item, new coupon) and search.
  - `Main Grid` -> KPIs at top, charts area, lists below.
  - `Wider panels` for detailed tables: orders table, reservations table, users table.
- Tables: responsive design with stacked rows on small screens or show a simplified card view.
- Use `sticky` column headers for long tables and pagination controls.

J) Auth pages (Login / Signup / Reset)
- Lightweight centered cards: `max-w-md mx-auto py-12`.
- Include social login buttons (optional) and password strength meter.

K) 404 / Error Fallback
- Friendly 404 page with search input and CTAs to homepage or contact support.
- ErrorBoundary fallback: small card with an apology, a reload button and a link to report a bug.

6) Accessibility & ARIA Guidelines
- Landmark roles: ensure `header`, `main`, `nav`, `footer` elements are used.
- Skip link: top-level `a.skip-link` to `#main` for keyboard users.
- Focus visible: custom focus styles (`box-shadow` ring) but maintain OS default for clarity.
- Buttons/links: include `aria-label` when text is not descriptive (e.g., icon-only buttons).
- Forms: each input with associated `label` using `for` + `id`. For complex widgets, include `aria-describedby`.
- Alerts/toasts: use `role="status"` for non-critical notifications and `role="alert"` for errors; ensure they are reachable by screen readers.
- Modals: trap focus inside modal, return focus to trigger element on close, and mark background inert when modal open (use `aria-hidden` on background content).
- Tables: provide descriptive `caption` and `aria-label` when necessary. Use row headers with `th scope="row"`.

7) Responsive Behavior & Breakpoints
- Default breakpoints: `sm(640)`, `md(768)`, `lg(1024)`, `xl(1280)`.
- Layout patterns:
  - Mobile-first single-column stacking.
  - At `md` convert to two-column for content-rich pages.
  - At `lg` convert to multi-column grids for galleries, dashboards.
- Navigation:
  - Mobile: hamburger -> slide-in menu or full-screen menu.
  - Desktop: top nav with inline links and dropdowns for account/menus.
- Admin tables: collapse into cards at `sm`/`md` thresholds for better readability on small devices.

8) Images, Media, Fonts & Assets
- Image formats: use `webp` where possible, fallback to `jpg/png`.
- Sizes: serve responsive images with `srcset` or use `<picture>` for art direction.
- Lazy loading: add `loading="lazy"` on images off-screen.
- Avatar: use 48px / 64px sizes depending on context; use `object-cover`.
- Fonts: load from local or Google Fonts with `display=swap`.
- Preconnect to font host: `<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>`.
- Assets folder: `/public/assets/menus/`, `/public/assets/gallery/`.

---

9) Animations & Motion Guidelines
- Keep motion subtle and purposeful.
- Use `transition` for hover states: `transition transform opacity` with `duration-150`.
- Use `framer-motion` for entrance/exit animations of large components (e.g., route transitions, modals).
- Motion reduction: respect `prefers-reduced-motion` media query to disable non-essential animations.

Example subtle hover for cards:
- `transform-gpu hover:-translate-y-1 hover:shadow-lg`.

10) Print Styles (Receipt / Bill)
- Create `.print-receipt` class used for receipts page; print stylesheet includes `@media print` rules.
- Hide navigation and interactive elements via `.no-print`.
- Use compact fonts and layout, single-column receipt width (e.g., 80mm equivalent in px), ensure `page-break-inside: avoid;` for critical blocks.

Receipt header example
- Business name (bold), address, date/time, order number, items list, subtotal, tax, total (bold), thank you note.

---

11) Dark Mode & Theming
- Two approaches:
  - `prefers-color-scheme` automatic dark adjustments.
  - Manual toggle (persist choice in `localStorage` and apply `class="dark"` to `html` or `body`).
- Tailwind: enable `darkMode: 'class'` and define `dark:` variants for colors.
- Example: `.bg-card` -> `bg-white dark:bg-[#0b1220]`.

---

12) Performance & Lighthouse Tips
- Use image optimization (WebP, responsive images).
- Avoid large JS bundles: split vendor and route-level bundles.
- Use lazy loading for routes and images.
- Minimize fonts: choose 1-2 families and limit weights.
- Enable caching headers and service worker for offline PWA behavior.
